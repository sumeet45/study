Quick Start - 

1) Creating and nesting components -
  * React apps are made out of components. A component is a piece of the UI (user interface) that has its own logic and appearance. A component can be as small as a button, or as large as an entire page.
  * React component names must always start with a capital letter, while HTML tags must be lowercase.
  * The "export default" keywords specify the main component in the file.
       export default function MyApp() {
          return (
            <div>
              <h1>Welcome to my app</h1>
              <MyButton />
            </div>
          );
        }


2) Writing markup with JSX  -
  * Component written using JSX. JSX is stricter than HTML. You have to close tags like <br />. Your component also can’t return multiple JSX tags. You have to wrap them into a shared parent, like a <div>...</div> or an empty <>...</> wrapper.

3) Adding styles  -
  * In React, you specify a CSS class with className.
            <img className="avatar" />

4) Displaying data -
 * "{user.name}" using this, we can add variable in JSX.
 * To add attribute value, we can use like following -   <img className="avatar" {user.imageUrl} />
 * You can put more complex expressions inside the JSX curly braces too, for example, string concatenation. {"Photo" + user.name}

5) Conditional rendering  -
 * you can use an if statement to conditionally include JSX.
 * If you prefer more compact code, you can use the conditional ? operator. Unlike if, it works inside JSX. -  
      {isLoggedIn ? <AdminPanel /> : <LoginPanel />}
 * When you don’t need the else branch, you can also use a shorter logical && syntax:
      {isLoggedIn && <AdminPanel />}

6) Rendering lists -
 * using for loop or array's map function, we render list in React.
 * we will add key attribute to <li>. it will contain unique value, like db Id. React uses your keys to know what happened if you later insert, delete, or reorder the items.

7) Responding to events  -
 * You can respond to events by declaring event handler functions inside your components.
        <button onClick={handleClick}> Click Me </button>

8) Updating the screen  -
 * use "useState" hook to add state to component.
    function MyButton(){
       const [val, setVal] = useState(0);
       const handleClick = () => {
          setVal(val => val + 1)
       }
       return (
         <button onClick={handleClick}>
           Clicked {val} times
         </button>
       )
    }

9) Using Hooks -
 * You can also write your own Hooks by combining the existing ones.
 * You can only call Hooks at the top of your components (or other Hooks). If you want to use useState in a condition or a loop, extract a new component and put it there.

10) Sharing data between components -
 * you’ll need components to share data and always update together.
 * To share data, you need to move the state from the individual buttons “upwards” to the closest component containing all of them.
 * Also, we need to share state with child components using props.
 * ex.

 import { useState } from 'react';

export default function MyApp() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h1>Counters that update together</h1>
      <MyButton count={count} onClick={handleClick} />
      <MyButton count={count} onClick={handleClick} />
    </div>
  );
}

function MyButton({ count, onClick }) {
  return (
    <button onClick={onClick}>
      Clicked {count} times
    </button>
  );
}

Thinking in React -

* When you build a user interface with React, you will first break it apart into pieces called components. Then, you will describe the different visual states for each of your components. Finally, you will connect your components together so that the data flows through them.

Start with the mockup -

1) Step 1: Break the UI into a component hierarchy -
  * Start by drawing boxes around every component and subcomponent in the mockup and naming them.
  * Depending on your background, you can think about splitting up a design into components.

2) Step 2: Build a static version in React -
  * To build a static version of your app that renders your data model, you’ll want to build components that reuse other components and pass data using props. You can choose "top to bottom" or "bottom to top"

3) Step 3: Find the minimal but complete representation of UI state -
  * To make the UI interactive, you need to let users change your underlying data model. You will use state for this.
  * decide which should be state based on following.
  * Which of these are state? Identify the ones that are not:
      1) Does it remain unchanged over time? If so, it isn’t state.
      2) Is it passed in from a parent via props? If so, it isn’t state.
      3) Can you compute it based on existing state or props in your component? If so, it definitely isn’t state!
    What’s left is probably state.

4) Step 4: Identify where your state should live -
  * After identifying your app’s minimal state data, you need to identify which component is responsible for changing this state, or owns the state.
  * Find component, where state should be available. suppose, that state require in 1 place only, then use it in that component. If state required in different component, then use it in common parent, so you can pass it to child component.

5) Step 5: Add inverse data flow -
  * using Functions, which we pass as props, we can update state in parent component and we can achieve inverse data flow.
