
* Database - It is a simple container to store data. But instead of just dumping files into folders, the database organized the data. so it is easy to access, to manage and to search.
* If we use files instead of DB, it will be quite messy and a long process. 
* Database can handle huge amount of data. It is safe and secure to store important and critical data inside the database.
* SQL is a language to talk with a database. stands for Structured Query Language.

1) DBMS & SQL Server -
* We have data inside the database. With different SQL, every user accesses the database.
* DBMS is software which manages requests to the database for every user, also it is going to make the priority which SQL must be executed first and also handles security.
* SERVER - server means a system which should be online 24/7. It hosts our database and DBMS software.

In short -
Database - stores data
SQL - speaks to the database
DBMS - manages the database
server - where the database lives

2) DB types - 
* relational DB - in it, we have tables, which contain rows and columns. They have a relationship with other tables also. Hence we call this relational DB
* key-value  db - key-value db is also called a no-relational db.
* column based db - in it, data is grouped according to the column. It is quite advanced and handles big data. The main purpose of this DB is to search for data.
* graph based DB - main focuse here is the relationship between objects. So the main idea here is how to connect data points.
* Document-Based DB - The data is stored as entire documents where the structure of data is not that important.

* From above, key-value, column based, graph, document-based DBs, are called "No-SQL database".
* Microsoft SQL server, MySQL, PostgreSQL are types of SQL relational Database.
* key-value DB example - Redis, Amazon Dynamo DB
* column-based DB example - Apache Casandra, Amazon Redshift
* Graph DB example - Neo4J
* Document-Based DB example - MongoDB


3) Database Structure -
* The databases are very structred and organized. It has the following hierarchy.
hierarchy =>
* Server - Who holds databases and will be available online all time.
* Database - In Server, we can have multiple databases.
* Schema - In each Database, we can have multiple schema. It is like category, which helps to group up related objects.
* Table - Each Schema contain multiple objects like tables. In table, we have column and row. We call "field" for column. For each row, we call it as "record". We also have "Primary key" in table. It helps to find unique record.
* Cell - single cell store value which have specific data types. Data types -> Numeric (INT, DECIMAL), Text (CHAR, VARCHAR), Date & Time (DATE, TIME)


4) SQL Commands -
* DDL commands - helps to create table, alter table or drop table.
* DML commands - helps to insert a new data, update and delete data.
* DATA Query Language - Using 'Select', we can retrieve data.

7) Query Data (SELECT)
* Normally, data is available in the table & table is available in Database. So whenever we want data, we will use SELECT query to grab data. Using this, we can ask for data from database.
* It is called reading from database. Select statement help here to read from database.
* SQL query have multiple clauses - SELECT, DISTINCT, TOP, FROM, JOIN, WHERE, GROUP BY, HAVING, ORDER BY.
* Query - Select * ( select * is going to go and retrieve all the columns from your table.)
* From Clause - tell SQL where to find your data. So with Select, we select the columns that we want and the "From", you specify the table where your data come from.
* "SELECT * From User" - 
    In this query, SQL execute first "From" Clause. Its going to go and retrieve all the data from the database to the results.
    Second step, SQL check "SELECT" statement. It decide which columns we have to keep in the result.

9) SQL Query Where (Filter your data) -
* we use where to filter our data based on condition, or any data that fulfuill condition going to stay in the output in the result.
* syntax - SELECT * FROM Table WHERE condition
* Query - 
SELECT name, country from customers where score != 0
SELECT * from customers where country = 'Germany'


10) SQL query ORDER By (Sort your data) -
* It helps to sort data. With this, we can do ascending or descending
* while sorting, we have to spacify two things. 1) Ascending or Descending, 2) Table
* by default, it is ascending. if we don't write ascending, it will take ascending. so better keep habit of adding ascending or descending. 
* example - 
select * from customers order by Score DESC
